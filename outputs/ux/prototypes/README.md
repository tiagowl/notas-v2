# Protótipos — Notas v2

Especificação de **protótipo de interação** (comportamento, estados e transições) para implementação em Next.js + Chakra UI v3.

> Não há arquivos Figma neste output — o protótipo é documental e serve como spec funcional para o Frontend Dev.

## Índice

| Arquivo | Conteúdo |
|---------|----------|
| [user-flows.md](./user-flows.md) | Fluxos de usuário detalhados |
| [interactions-states.md](./interactions-states.md) | Estados da UI, transições, feedback |

## Como usar

1. Implementar fluxos na ordem de `user-flows.md` (MVP primeiro).
2. Para cada componente, consultar estados em `interactions-states.md`.
3. Validar com roteiro em `../usability-reports.md` após Sprint 3.
